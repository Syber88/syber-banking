# Documentation

This folder contains the documentation for the Syber Banking API.

| Document | Purpose |
| --- | --- |
| [Architecture](architecture.md) | Describes the application structure and design decisions. |
| [API documentation](api.md) | Describes endpoints, request data, responses, and errors. |
| [Database documentation](database.md) | Describes the data model and database behavior. |
| [Setup guide](setup.md) | Describes how to prepare and start the service. |
| [Deployment guide](deployment.md) | Gives a checklist for a production deployment. |
| [Testing guide](testing.md) | Describes test commands and test improvements. |
| [User manual](user-manual.md) | Describes how to use the API in Swagger UI. |

Architecture Decision Records (ADRs) are in [adr](adr). They describe the reason for the main technical decisions.

## Writing standard

Write new technical documentation in ASD-STE100-style controlled English. Use short sentences. Use active voice. Use one meaning for each technical term. Give one action in each numbered step. Use `WARNING` text when an action can cause data loss or a security problem.
